## `<Tabs>`和`<TabItem>`

首先介绍一下`<Tabs>`组件,还是一个挺好用的组件,下面是它的简单示例:

```jsx title="example.mdx" {1,2,5-7} showLineNumbers
import Tabs from "@theme/Tabs"
import TabItem from "@theme/TabItem"
```

### `groupId`

当你的mdx文件中有多个`<Tabs>`组件时,你可以使用`groupId`将它们进行关联,`groupId`相同的`<Tabs>`就会自动关联关联的用处就是当一个.例如你现在有tab1和tab2两个选项卡，它们的值的情况如下:

```jsx title="groupId" showLineNumbers
<Tabs groupId="fruit">
    <TabItem value="apple" label="苹果" defualt>这是一个苹果🍎</TabItem>
    <TabItem value="banana" label="香蕉">这是一个香蕉🍌</TabItem> 
    <TabItem value="orange" label="橘子">这是一个橘子🍊</TabItem>
</Tabs>

// 它们只是第三个<TabItem>不一样
```

## `code blocks`

现在再来讲一讲`code blocks`,也就是代码块,与基本的`markdown`一样使用三个反引号来表示代码块,但是`docusaurus`有所拓展,如下:

- language : 指定变成语言,将代码渲染成这个语言的样式(不是所有的编程语言都支持)
- title : 用来制定代码块的标题
- highlighting : 使代码块中的某些代码高亮
    - metadata string : 使用`{1,3-5}`这种形式来高亮
    - magic comments : 使用魔力注释来高亮
- showLineNumbers : 显示行号

````
```js title="显示行号" showLineNumbers {2-4,6}
const bing = {
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")
```
````

```js title="显示行号" showLineNumbers {2-4,6}
const bing = {
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")
```

### 使用注释高亮代码

你也可以使用注释来高亮对应的代码行:
- highlight-next-line : 高亮下一行代码
- highlight-start, highlight-end : 高亮这两句注释所包含的代码块

````
```js title="显示行号" showLineNumbers
const bing = {
    // highlight-next-line
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")

// highlight-start
const example = {
    name: 'example',
    age: '?',
    hobby: '?'
}
// highlight-end
```
````

```js title="显示行号" showLineNumbers
const bing = {
    // highlight-next-line
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")

// highlight-start
const example = {
    name: 'example',
    age: '?',
    hobby: '?'
}
// highlight-end
```

:::warning
不同语言的注释是不一样的,比如`python`的注释是`#`,错误的注释将不会起到该有的作用,请注意这一点
:::

:::tip
建议使用注释的这种方式来高亮代码,因为这样你就不必去数代码的行数,并且它也不会受代码行数的改变而受到一些影响
:::

### 自定义魔力注释

想要自定义的魔力注释(magic comments),你需要修改`docusaurus.config.js`配置文件,在`themeConfig.magicComments`数组中新增你的魔力注释对象:

- className : 指定魔力注释的样式
- line : 注释生效的语句,就好比你要高亮,你就要写`highlight-next-line`这句话
- block : 指定一个对象,它有两个键
    - start : 起始位置的语句
    - end : 终止位置的语句

<Tabs>
    <TabItem value='new magic comment' label="docusaurus.config.js">
    
    ```js
        themeConfig: {
            magicComments: [
                {
                    className: 'theme-code-block-highlighted-line',
                    line: 'highlight-next-line',
                    block: {start: 'highlight-start', end: 'highlight-end'},
                },
                {
                    className: 'code-block-error-line',
                    line: 'this will error',
                    block: {start: 'error-start', end: 'error-end'},
                }
            ]
        }
    ```

    </TabItem>

    <TabItem value="example.md" label="example.md">
    
    ````
    ```js 
    const num = 3
    // this will error
    num = 4
    // 常量的值不能改变

    // error-start
    let noMoney = true
    if(noMoney === true) {
        console.log('我没钱了')
    } else {
        console.log('我有钱了')
    }
    // error-end
    ```
    ````

    </TabItem>

    <TabItem value="custom.css" label="custom.css">
    
    ```css 
    .code-block-error-line {
        background-color: #ff000020;
        display: block;
        margin: 0 calc(-1 * var(--ifm-pre-padding));
        padding: 0 var(--ifm-pre-padding);
        border-left: 3px solid #ff000080;
    }
    ```

    </TabItem>
</Tabs>

```js 
const num = 3
// this will error
num = 4
// 常量的值不能改变
// error-start
let noMoney = true
if(noMoney === true) {
    console.log('我没钱了')
} else {
    console.log('我有钱了')
}
// error-end
```

:::danger
在你的`docusaurus.config.js`配置文件中,最初没有`magicComments`字段的,当然也没有那个高亮的魔力注释啦,因为它是默认的,所以你可以不用写它,它也会生效.但是当你想添加自定义的魔力注释的时候,你需要将它写上,你不写的话,你的自定义魔力注释将会成为默认选项,你就没有原本高亮的魔力注释了,原本你使用高亮的魔力注释的地方将会实效,成为普通的注释,`metadata string`这种形式只会使用默认的魔力注释.
:::
