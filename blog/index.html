<Tabs>和<TabItem>​

code blocks​

现在再来讲一讲code blocks,也就是代码块,与基本的markdown一样使用三个反引号来表示代码块,但是docusaurus有所拓展,如下:

  • language : 指定变成语言,将代码渲染成这个语言的样式(不是所有的编程语言都支持)
  • title : 用来制定代码块的标题
  • highlighting : 使代码块中的某些代码高亮
    • metadata string : 使用{1,3-5}这种形式来高亮
    • magic comments : 使用魔力注释来高亮
  • showLineNumbers : 显示行号
```js title="显示行号" showLineNumbers {2-4,6}
const bing = {
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")
```
显示行号
const bing = {
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")

使用注释高亮代码​

你也可以使用注释来高亮对应的代码行:

  • highlight-next-line : 高亮下一行代码
  • highlight-start, highlight-end : 高亮这两句注释所包含的代码块
```js title="显示行号" showLineNumbers
const bing = {
    // highlight-next-line
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")

// highlight-start
const example = {
    name: 'example',
    age: '?',
    hobby: '?'
}
// highlight-end
```
显示行号
const bing = {
    name: 'moubing',
    age: '21',
    hobby: [coding, play games]
}
console.log("hello world")

const example = {
    name: 'example',
    age: '?',
    hobby: '?'
}

注意

不同语言的注释是不一样的,比如python的注释是#,错误的注释将不会起到该有的作用,请注意这一点

提示

建议使用注释的这种方式来高亮代码,因为这样你就不必去数代码的行数,并且它也不会受代码行数的改变而受到一些影响

自定义魔力注释​

想要自定义的魔力注释(magic comments),你需要修改docusaurus.config.js配置文件,在themeConfig.magicComments数组中新增你的魔力注释对象:

  • className : 指定魔力注释的样式
  • line : 注释生效的语句,就好比你要高亮,你就要写highlight-next-line这句话
  • block : 指定一个对象,它有两个键
    • start : 起始位置的语句
    • end : 终止位置的语句
  • docusaurus.config.js
  • example.md
  • custom.css
    themeConfig: {
        magicComments: [
            {
                className: 'theme-code-block-highlighted-line',
                line: 'highlight-next-line',
                block: {start: 'highlight-start', end: 'highlight-end'},
            },
            {
                className: 'code-block-error-line',
                line: 'this will error',
                block: {start: 'error-start', end: 'error-end'},
            }
        ]
    }
```js 
const num = 3
// this will error
num = 4
// 常量的值不能改变

// error-start
let noMoney = true
if(noMoney === true) {
    console.log('我没钱了')
} else {
    console.log('我有钱了')
}
// error-end
```

    background-color: #ff000020;

    margin: 0 calc(-1 * var(--ifm-pre-padding));
    padding: 0 var(--ifm-pre-padding);
    border-left: 3px solid #ff000080;
}

危险

在你的docusaurus.config.js配置文件中,最初没有magicComments字段的,当然也没有那个高亮的魔力注释啦,因为它是默认的,所以你可以不用写它,它也会生效.但是当你想添加自定义的魔力注释的时候,你需要将它写上,你不写的话,你的自定义魔力注释将会成为默认选项,你就没有原本高亮的魔力注释了,原本你使用高亮的魔力注释的地方将会实效,成为普通的注释,metadata string这种形式只会使用默认的魔力注释.

我现在来回顾一下这个网站是怎么部署的

首先是下载docusaurus，点击进入官网

npx create-docusaurus@latest my-website classic
// my-website只是一个目录名称，你可以任意修改
// classic是网站的风格，你也可以更改
信息

你需要安装node，并且版本不低于18.0

然后你可以需要进入该文件夹，使用cd my-website，docusaurus提供了很多方便快捷的命令：

npm start
npm run build
npm run deploy
// 还有很多，你可以到package.json中查看

现在去github创建一个仓库

提示

仓库名称尽量取成<your github name>.github.io，因为这样设置的话，后面成功部署后的域名就是这个，比如我的这个网站的域名就是moubing.github.io。不然得话你的域名就是<your github name>.github.io/<repo name>，这样要长一节

接下来就是修改docusaurus的配置文件docusaurus.config.js，下面我将展示我的配置文件

const config = {
    // 网站首页的title，不知道是什么就自己改一下
    title: "bing'swebsite", 
    // 网站首页title下面的话
    tagline: 'bing are cool', 
    // 网站的图标，就是tab上的那个小鳄鱼
    favicon: 'img/favicon.ico',
    // 注意换成你的域名
    url: 'https://moubing.github.io/',
    // 设置网站的根路径，其实我也不太懂，管他的随便写了
    baseUrl: '/', 
    // 写你的github账号名称
    organizationName: 'moubing', 
    // 写你的仓库名称
    project: 'moubing.github.io', 
    // 以前的版本需要这句话，但是我看了官方文档，新的版本你可以不写，gh-pages是默认值
    deploymentBranch: 'gh-pages', 
    i18n: {
        // i18n全称internationalization，国际化的意思
        // 它会影响时间和其他东西的格式
        defaultLocale: 'zh',
        locales: ['zh']
    }

}
备注

还需要一个配置，你需要在你的电脑上设置一个环境变量GIT_USER，它的值设置为你的github账号名称（例如我的：moubing）

配置好后运行npm run deploy，它会先自动打包，将这个打包文件夹build上传到你的仓库，自动部署相关事宜

按理来说，这已经算是弄好了，你可以试着在浏览器上输入你的域名看看

这个很重要，我就是在这浪费了很多时间，还是细心一点

修改完docusaurus.config.js文件后记得保存，保存在进行后面的操作，我就是因为没有保存，一直在那瞎搞，原本最开始就弄的差不多了😖😖😖

当然还可能遇到一些问题，比如：

Failed to connect to github.com port 443 after 22130 ms: Couldn't connect to server

这个问题多半是代理和防火墙导致的，它是暂时的，跟网络有关，你可以试着把代理或者防火墙关掉，其实还是看运气，遇到它怎么尝试都还是解决不了，我的建议是过一会再来试一试

Docusaurus blogging features are powered by the blog plugin.

Simply add Markdown files (or folders) to the blog directory.

Regular blog authors can be added to authors.yml.

The blog post date can be extracted from filenames, such as:

  • 2019-05-30-welcome.md
  • 2019-05-30-welcome/index.md

A blog post folder can be convenient to co-locate blog post images:

The blog supports tags as well!

And if you don't want a blog: just delete this directory, and use blog: false in your Docusaurus config.
